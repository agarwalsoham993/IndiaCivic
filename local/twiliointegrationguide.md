Since the application is deployed through Google AI Studio's "Publish App" option (which compiles and hosts it on Google Cloud Run), the process of starting the live WhatsApp service is extremely straightforward and doesn't require any manual server configuration or proxy servers (like ngrok) in production.

Here are the concrete steps to go live:

---

### Step 1: Deploy / Publish the App on Google AI Studio
1. Open your project in the **Google AI Studio UI**.
2. Click the **Publish App** option.
3. Once the deployment finishes, copy the public URL generated by Google AI Studio for your hosted applet (e.g., `https://indiacivic-applet-xyz.a.run.app`). 

---

### Step 2: Configure Environment Variables in AI Studio
Inside the Google AI Studio publisher console, navigate to the **Secrets** or **Environment Variables** panel and add your WhatsApp Business provider credentials:
* `TWILIO_ACCOUNT_SID`
* `TWILIO_AUTH_TOKEN`
* `TWILIO_WHATSAPP_NUMBER`

*(These are automatically injected into `process.env` in the Cloud Run container at runtime).*

---

### Step 3: Link the Webhook Callback
Go to your WhatsApp BSP console (e.g. Twilio Console):
1. Navigate to **Messaging > Sandbox settings** (or Production Sender settings).
2. For the incoming message webhook URL, enter your Google AI Studio Cloud Run URL appending our route:
   ```
   https://<your-applet-id>.a.run.app/webhook/whatsapp-trigger
   ```
3. Set the HTTP request method to `POST`.

---

### Step 4: Outbound Message Sending Logic
To let the server send replies back directly, the Express handler in `server.ts` uses the `twilio` package (which we installed) to dispatch messages to the user:

```typescript
import twilio from "twilio";

// Initialize twilio client using runtime environment variables injected by AI Studio
const twilioClient = twilio(
  process.env.TWILIO_ACCOUNT_SID,
  process.env.TWILIO_AUTH_TOKEN
);

// Call this function inside server.ts webhook replies to push real messages to the user
async function sendOutboundWhatsApp(to: string, bodyText: string) {
  try {
    await twilioClient.messages.create({
      from: `whatsapp:${process.env.TWILIO_WHATSAPP_NUMBER}`,
      to: `whatsapp:${to}`,
      body: bodyText
    });
    console.log(`[WhatsApp Outbound] Sent to ${to}`);
  } catch (error) {
    console.error("[WhatsApp Outbound Error] Failed to send:", error);
  }
}
```

Since the environment is fully managed by Google Cloud Run, it scales automatically, connects to the firestore rule sets, and remains active 24/7 without developer maintenance.
